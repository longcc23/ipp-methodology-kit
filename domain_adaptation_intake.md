## 5. 方法路径选择（勾选 + 一句理由）

| 路径 | 是否采用 | 理由 |
|------|----------|------|
| QEC + LLM 结构化编码 | | |
| 向量聚类（如 BERTopic） | | |
| LLM 语义归纳（多组独立） | | |
| 双路交叉验证 | | |
| 纯人工编码样本集（金标） | | |
| 迭代编码（先形式/结构 → 后内容/行为） | | |

> **迭代编码说明**：若数据类型较新（如模拟对话、工单日志），初始编码方案可能不适配。可先做一轮形式编码验证数据可分析性，再设计内容编码方案。两轮之间必须做**预测试**确认新方案的覆盖率与区分度，并将方案变更记录为 ADR。

> **聚类路径说明**：聚类走单路还是双路，取决于 Code 粒度——在 Step 2 之后才能判断。Code 粒度高（唯一率 > 90%）→ 单路即可；Code 粒度低或 BERTopic 产出超大主题（> 50% 聚入同一主题）→ 需双路聚类。详见 `README.md`「决策点 3」。
