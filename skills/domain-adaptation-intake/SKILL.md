# 新数据源准入与方法评估

## 何时调用

- 拿到一批新类型的数据，想用 QEC 方法分析
- 需要判断「这批数据能不能做 QEC、该怎么适配」

## 执行步骤

1. **先看数据**：读取样本数据（至少 5 条），理解数据结构、字段含义和内容特征。
2. **填写准入问卷**：基于数据样本，填写 `methodology-kit/domain_adaptation_intake.md`。关键空项（分析单位、研究问题、已知偏差）必须补齐。
3. **评估三个关键问题**：
   - **能否做 QEC？** 数据是否包含可引用的原文片段（Q）？是否允许研究者做推断（E）？能否归纳为可聚合标签（C）？
   - **需要哪些适配？** 对照 `README.md` 决策点 1（E/C 操作定义适配）：数据的对话/评价/访谈/日志属性，决定了 E 必须额外做什么、C 有什么特殊约束。
   - **有没有结构性陷阱？** 对话/交互类数据：验证 speaker 时序是否与存储顺序一致。多来源数据：检查样本筛选标准的方法论含义。
4. **规划方法路径**：对照 `domain_adaptation_intake.md` 第 5 节，勾选适用的路径。若数据类型较新，评估是否需要迭代编码（先形式编码验证可分析性，再做内容编码）。
5. **保存产出**：将填好的文件保存到项目目录（如 `docs/domain_adaptation_intake.md`），并在项目 README 中链接。

## 与仓库的衔接

- 参考本工具包的 `README.md`「通用流程与关键决策点」
- 若同一仓库内有已跑通的案例研究，可在 intake 中引用其方法论文档作为对照

## 输出

- 一份填完的 `domain_adaptation_intake.md`
- **可行性判断三句话**：
  1. **能不能做**：QEC 是否适用，不适用的话建议用什么替代方法
  2. **先试点还是全量**：基于数据量和不确定性给出建议
  3. **最大风险是什么**：最可能导致分析失败的因素（如数据结构陷阱、偏差、伦理问题）
